# 桌面数据分析器第二阶段设计说明

- 日期：2026-05-03
- 主题：界面优化与 Windows exe 打包交付
- 目标：在现有第一版可运行源码基础上，提升界面成熟度和操作体验，并交付可复现的目录版 `.exe`

## 1. 第二阶段定位

第二阶段不是重做产品，而是在第一版已经打通主流程的基础上，做一次**中度产品化升级**。

本阶段的成果应满足两个目标：
- 打开程序后，整体观感和交互更接近正式桌面软件
- 代码仓库可以稳定打包出一个可分发的目录版 Windows `.exe`

这轮工作不追求引入大量新功能，而是优先提升现有能力的可用性、可理解性和可交付性。

## 2. 为什么选择这个范围

围绕“优化界面后打包 exe”，有三种常见推进方式：

### 方案 A：轻量打磨 + 稳定打包
只对布局、提示、默认行为做最少量修正，然后尽快完成打包。

优点：
- 风险最低
- 最快形成交付件

缺点：
- 虽然可交付，但产品感提升有限
- 用户打开后仍能明显感觉到这是偏工程原型的界面

### 方案 B：中度产品化升级
在现有第一版上，补足桌面软件最关键的体验层：更清晰的主界面层次、状态反馈、空状态、重置动作、导入后的默认行为、导出反馈，以及稳定的目录版打包流程。

优点：
- 视觉和体验都会明显提升
- 范围仍可控，不会拖慢交付节奏
- 最符合“先把效果做好，再交付 exe”的目标

缺点：
- 实现量高于轻量打磨
- 需要少量界面结构调整

### 方案 C：强产品感重做
大幅重做视觉风格和交互结构，追求更强产品感。

优点：
- 观感提升最大

缺点：
- 容易把范围做大
- 会显著延后 exe 交付

最终选择：**方案 B：中度产品化升级**。

## 3. 本阶段范围

### 3.1 必做内容
- 优化主窗口布局与层次
- 增加状态栏和更清晰的操作反馈
- 增加“重置工作区”能力
- 优化左侧配置区分组与默认值
- 优化空状态显示
- 优化导入后的默认展示行为
- 优化导出完成后的成功提示
- 增加 PyInstaller 打包配置
- 产出目录版 `.exe`
- 补充打包与运行命令文档

### 3.2 允许的小幅增强
本阶段允许加入少量高价值增强，但必须满足“低风险、高收益、服务于现有主流程”的原则。允许范围包括：
- 导入文件后自动展示源文件名、工作表名和行列统计
- 在没有可画图数据时显示明确提示，而不是空白区域
- 自动选择更合理的默认图表字段
- 导出成功后在状态栏显示文件位置
- 表格和图表区域的标题或分组更明确

### 3.3 暂不纳入本阶段
以下内容不进入本阶段：
- 多工作表切换器
- 可保存/加载处理方案
- 高级公式编辑器
- 多图表并列布局
- 拖拽式分析流程
- 安装程序制作
- 自动升级能力

## 4. 界面优化设计

### 4.1 总体目标
当前第一版已经具备工作台结构，但界面仍偏工程实现。本阶段要把它提升为“看起来更像正式桌面软件”的版本。

优化方向：
- 让界面分区更清楚
- 让主操作更容易被看到
- 让无数据、导入后、处理后、导出后这几种状态更明确
- 让用户在不看说明的情况下也能理解主流程

### 4.2 主窗口结构
主窗口仍保留三栏结构，但会优化为更明确的工作台布局：

- **顶部工具栏**
  - 导入 Excel
  - 执行处理
  - 导出结果
  - 重置工作区

- **左侧配置面板**
  - 数据源信息卡片
  - 数据处理配置卡片
  - 图表配置卡片
  - 执行与导出操作区

- **中间内容区**
  - 原始数据 / 处理结果标签页
  - 表格区增加更清晰的页签命名与空状态

- **右侧图表区**
  - 图表标题
  - 无可用图表时显示提示信息
  - 有数据时显示默认图表

- **底部状态栏**
  - 显示当前状态：未导入 / 已导入 / 已处理 / 已导出
  - 显示最近一次操作结果

### 4.3 关键交互改进
1. **导入后**
   - 自动更新数据源信息
   - 自动填充列选择框
   - 如果存在合适字段，自动生成默认图表

2. **执行处理后**
   - 自动切到“处理结果”页签
   - 状态栏显示处理成功
   - 图表尽量保持与当前结果同步

3. **导出后**
   - 状态栏显示导出成功及目标路径
   - 必要时弹出轻量成功提示

4. **重置工作区**
   - 清空当前数据、表格、图表和配置输入
   - 回到程序初始状态

## 5. 技术设计

### 5.1 UI 层调整方向
在不推翻现有结构的前提下，对以下文件进行增强：
- `ui/main_window.py`：增加状态栏、重置逻辑、默认行为和更清晰的工作台结构
- `ui/panels.py`：增强数据源分组、操作按钮布局、空状态相关控件
- `services/chart/render.py`：补充无图表数据时的提示绘制辅助逻辑
- `app/state.py`：补充更完整的界面状态字段

### 5.2 打包设计
使用 **PyInstaller** 生成目录版 Windows 应用。

本阶段应新增：
- `build/` 或 `scripts/` 中的打包脚本
- `desktop-data-analyzer.spec` 或同类 PyInstaller 配置文件
- 必要的资源收集配置
- `dist/` 输出目录验证流程

### 5.3 打包目标
目录版交付物应满足：
- 双击主程序可启动
- 不依赖开发环境命令行手工介入
- 能读取本地 Excel
- 能导出结果 Excel
- 图表与界面正常显示

## 6. 验收标准

本阶段完成后，应满足以下条件：
- 主窗口整体观感明显优于第一版
- 工具栏、左侧配置区、中间表格区、右侧图表区、底部状态栏职责清晰
- 支持重置工作区
- 导入、处理、导出各阶段均有明确反馈
- 默认图表行为合理，空状态不再生硬
- 能通过测试验证核心逻辑
- 能成功打包出目录版 `.exe`
- 打包产物能够在本机直接启动运行

## 7. 风险与控制

### 7.1 主要风险
- 为了追求“更好看”而过度改动结构，影响稳定性
- PyInstaller 漏收依赖，导致打包后图表或 Qt 运行异常
- 过多加入新功能，拖慢交付

### 7.2 控制策略
- 只在现有结构上增强，不重做整体架构
- 打包优先选择目录版，降低单文件模式的兼容风险
- 优先确保主流程稳定，其次再做观感优化
- 打包后必须做一次本机真实启动验证

## 8. 实施边界

这份设计对应的是“第二阶段：界面优化 + exe 打包交付”，不是完整商业化版本。它的目标是让当前第一版从“可运行原型”进入“更成熟、可直接演示和交付”的状态。
